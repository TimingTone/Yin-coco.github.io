python异步IO与协程

Posted on 2019-12-18 • In python • • Heat: ℃

在python高级IO编程中，为了解决多线程的线程间同步，同步编程的并发性太低，以及回调模式编码复杂度高等问题，需要引入异步IO和协程。

异步IO

同步与异步

  • 同步： 同步就是发起一个调用后，被调用者未处理完请求之前，调用不返回。

  • 异步： 异步就是发起一个调用后，立刻得到被调用者的回应表示已接收到请求，但是被调用者并没有返回结果，此时我们可以处理其他的请求，被调用者通常依靠事件，回调等机制来通知调用者其返回结果。

同步和异步的区别最大在于异步时调用者不需要等待处理结果，被调用者会通过回调等机制来通知调用者其返回结果。

yield

Python对协程的支持是通过generator实现的。

在generator中，我们不但可以通过for循环来迭代，还可以不断调用next()函数获取由yield语句返回的下一个值。

yield from

python3.3新增了yield from语法,其在实现异步IO中尤为重要.

基本用法:

  • yield from (iterable)其后对象可以为list, dict, generator, 协程.

  1. main 调用方 g2委托生成器 gen　子生成器

  2. yield from 会在调用方和子生成器之间直接建立双向通道, 而非利用g2建立链接的跳板.

协程(Coroutine)

python为了将语义变得更加明确,　就引入了async和await关键词用于定义原生的协程.

使用原生的协程不能用next(None)代替.send(None),　只能用.send(None)方式调用.

注意:async定义的函数不能定义yield, await也无法应用于非async定义的函数里, 可应用＠types.coroutine定义非async定义的生成器实现协程.
